The headline body of work in this session — and the most important piece for you to preserve — is the **click recorder and replay system**. Vaughn wanted to be able to drive a demo manually, click-by-click, then copy the transcript verbatim, paste it into a script file, and replay it. As of CMD100.83 that loop works end-to-end across navigation, tab activation, prospect open/close/edit, kanban drag-and-drop, form-field entry on text/select/date/pill inputs, sentinel-driven add-new-company flows, form submission, and cross-session settle (`page_ready` event). The architecture is a global capture-phase click listener in hud-shell.js that walks up from event.target, classifies the element by container context (nav/tab/subtab/dialog) plus per-module hardcoded rules (`.p-card` → Open Prospect, `.add-card-btn` → Form Open "Add Prospect", etc.), extracts a clean label stripped of icon glyphs and badge counts, and broadcasts a `recorder_event` envelope on the shared Compass channel. Aegis subscribes, deduplicates by event_id across the dual-channel receipt path, and appends to its transcript when the corresponding session is "armed" via the per-row record button on the Sessions panel. Replay runs the same lines through the cmd-center COMMANDS table; navigation verbs use the `page_ready` settle protocol — receiver broadcasts when both channels are subscribed, dispatcher waits for the timestamp to advance — so subsequent commands don't fire into a tearing-down WebSocket.

Pipeline is the most recent surface to have received deep instrumentation. The Add Prospect drawer carries the `Add Prospect` title (renamed from `New prospect` for `Form Open` symmetry), uses `— Select —` placeholder defaults on Timeline urgency / Priority / Assigned-to so the recorder captures actual user choices, has a `+ Add new company…` sentinel option that triggers a `Form Add` consolidation in the recorder (single line instead of two), and uses the shared `PersonPicker` component for the Assigned-to field (grouped by department, color-coded avatars, search, External-last sort). The kanban itself supports HTML5 drag-and-drop with optimistic UI updates and PATCH rollback on failure; the recorder captures drops as `Move Prospect "<title>" "<stage>"`. The prospect-detail descended view has Back / Edit buttons that round-trip to the kanban and re-open the drawer in edit mode pre-populated. Currency was switched from ¥ to $ throughout. Card layouts had multiple visual passes — 2-line title clamp with min-height for vertical alignment, meta-row min-height to handle missing tags/values, valuation bumped to 17px as the most-important-element-per-card, full names instead of first names, dates with year.

A few traps to avoid. **Module-scope state leaks into closures.** The recorder's `_recordArmed` map and the settle protocol's `_pageReadyTs` map both started life inside the channel-setup IIFE and produced `ReferenceError` failures until hoisted to module scope. Any new map that the script runner needs to read must live at module scope in cmd-center.js. **CURRENT_USER is unreliable.** It's populated by hud-shell's API path and silently fails on pages with different api.js wiring. The trusted source is `window._aegisSelf`, mirrored from cmd-center's resolved `_mySession`. The recorder reads `_aegisSelf` first. **Class regexes with `\b` boundaries match hyphenated extensions.** `\bp-card\b` matches `p-card-name`. Use `el.classList.contains()` for strict token matching. **Vaughn manages `js/version.js` externally.** Don't bump unprompted; ask which stamp to use after each delivery. **The schema inventory is authoritative; ad-hoc probing is not.** When you need a column shape, ask Vaughn to query Supabase directly via SQL Editor — he's fast at it.

## 5. Recorder + Replay System (CMD100.50ŌĆōCMD100.83)

This is the largest body of work in this session ŌĆö and the most important to preserve.

### 5.1 Architecture

**Click recorder** (in `hud-shell.js`, function `_recorderInit`):
- Global `document.addEventListener('click', ..., true)` capture-phase listener.
- Walks up from `event.target` via `findActionable()` to find the nearest "actionable" element.
- Classifies via `classify()` to produce a canonical command string.
- Extracts label via `extractLabel()` ŌåÆ `_textWithoutBadges()` ŌåÆ `_cleanLabel()`.
- Broadcasts `recorder_event` envelope with `{event_id, user_id, alias, command, ts}` via `window._cmdCenterChannelSend`.

**Form-input recorder** (also in `hud-shell.js`):
- Capture-phase `blur` listener for `<input>`/`<textarea>` ŌåÆ `Form Insert "<label>" "<value>"`.
- Capture-phase `change` listener for `<select>` ŌåÆ `Form Select "<label>" "<visible-text>"`.
- Per-field `WeakMap` cache (`_lastFieldValue`) for duplicate suppression.
- Empty-value suppression (no emit on blur with empty value).
- **Sentinel handling** (CMD100.76): when a `<select>` change goes to an option whose value matches `/^__add_new_.+__$/`, the emit is suppressed and `el.__hudAddPending` is set; the next change emits `Form Add "<field>" "<new value>"` instead of `Form Select`.

**Recorder receiver** (in `cmd-center.js`):
- Subscribes both channels to `recorder_event`.
- `_handleRecorderEvent` dedups via `_seenEventIds`, checks `_recordArmed[user_id]`, prepends alias and appends to transcript.
- Per-session "arm" toggle button on each session row in the Sessions panel (14├Ś14 hollowŌåÆfilled red dot). State in module-scope `_recordArmed = {}` map (ephemeral, cleared on Aegis reload).

### 5.2 Classification Rules (in `hud-shell.js` `classify()`)

In priority order:
1. **`data-aegis-cmd` override** ŌĆö element has explicit override, return verbatim.
2. **Pipeline-specific** (CMD100.55 / CMD100.65 / CMD100.78 / CMD100.80):
   - `.p-card` ŌåÆ `Open Prospect "<title>"` (title from `.p-card-name`)
   - `.add-card-btn` ŌåÆ `Form Open "Add Prospect"`
   - `.btn-back` ŌåÆ `Close Prospect`
   - `.btn-edit` ŌåÆ `Edit Prospect "<title>"` (title from `#hdr-title`)
3. **Pill chip in `.pill-group`** (CMD100.73) ŌåÆ `Form Select "<sec-label>" "<chip-text>"` (sec-label from preceding `.sec-label` sibling).
4. **Container-based**: walks up to 12 ancestors, sets flags `inNav`, `inTab`, `inSubtab`, `inForm`, `inDialog`, counts `tabDepth`.
   - `inNav && !inTab` ŌåÆ `Set Page "<label>"`
   - `inSubtab || tabDepth >= 2` ŌåÆ `Set SubTab "<label>"`
   - `inTab || role=tab` ŌåÆ `Set Tab "<label>"`
   - `inDialog && .btn-primary` ŌåÆ `Form Submit` (CMD100.76)
   - `inDialog || inForm` ŌåÆ `Click "<label>"`
   - `<a>` ŌåÆ `Set Page "<label>"`
   - default ŌåÆ bare label (top-level button)
5. **Fallback**: returns `null` (no emit).

### 5.3 Replay Verbs (in `cmd-center.js` `COMMANDS`)

**Navigation (NAV_VERBS ŌĆö get `page_ready` settle):**
- `Register <Module>` ŌĆö script-start primitive, navigates session to module landing.
- `Set Page <Module>` ŌĆö human-label alias for `Set View`.
- `Set View <key>` ŌĆö legacy lowercase view key.
- `Open Prospect <name>` ŌĆö finds `.p-card` by name match, clicks it.
- `Close Prospect` ŌĆö calls `window.closeProspect()` or navigates `/pipeline.html`.
- `Edit Prospect <name>` ŌĆö looks up id in `window.allProspects`, navigates `/pipeline.html?edit=<id>`.

**Tab activation:**
- `Set Tab <name>` / `Set SubTab <name>` ŌĆö registry-first via `AegisRegistry`, DOM-fallback via `_findTabInDOM` (queries `[role=tab]`, `.tab`, `.tab-btn`, `.subnav-item`, etc., with badge-stripped text comparison).

**Form replay (DOM-find fallback for inline drawers, iframe path for Compass forms):**
- `Form Open <name>` ŌĆö Compass form-defs registry first; DOM fallback finds `.add-card-btn` (Pipeline) or button matching name.
- `Form Insert <label> <value>` ŌĆö DOM helper `_findFormFieldByLabel` matches `.field > label` text.
- `Form Select <label> <value>` ŌĆö three paths in fallback:
  1. `<select>` field ŌĆö sets selectedIndex to matching option text, dispatches change.
  2. `.pill-group` matching section label ŌĆö clicks matching pill.
  3. **Hidden input + PersonPicker pattern** (CMD100.82) ŌĆö if hidden `<input>` matches AND `window.setAssignee` exists, looks up name in `window.pipelineResources` and calls `setAssignee(id, name)`.
- `Form Add <label> <value>` (CMD100.76) ŌĆö finds `<select>`, locates sentinel option, stubs `window.prompt` to return value, dispatches change to trigger page's existing add-handler, polls 5s for new option to appear.
- `Form Submit` ŌĆö iframe path first; DOM fallback clicks first `.btn-primary` inside open `.overlay.open` / `[role=dialog]` / `.drawer`.

**Pipeline-specific:**
- `Move Prospect <name> <stage>` ŌĆö looks up prospect in `window.allProspects`, optimistic-updates UI, PATCHes `prospects.stage`, rolls back on error.

### 5.5 Nav Settle Protocol (CMD100.63)

The single hardest problem in this session. **Do not regress this.**

Navigation commands tear down the WebSocket on the destination session before the ack can return. Without a settle protocol, the next command dispatches into a void.

**Solution:** receiver broadcasts `page_ready` event when `_onBothChannelsReady` fires (only on non-Aegis pages). Aegis stores the latest `page_ready` ts per user in `_pageReadyTs[userId]`. After dispatching a nav-class verb, Aegis snapshots `_preReadyTs`, then polls `_pageReadyTs[targetUserId]` until it advances, with an 8s deadline + 200ms server-side propagation margin.

Earlier attempts that failed:
- 1500ms fixed delay ŌĆö too short on slower loads, missed cmds.
- Heartbeat-freshness watch ŌĆö heartbeat cadence (~5s) didn't align with deadline; produced false-warning timeouts.

`page_ready` is the only reliable signal. Keep it.

## 11. Operating Principles Worth Preserving

- **Click-emit is the gold standard.** When a UI gesture should be replayable, the recorder should capture it without user-facing instrumentation. Manual scripting is the fallback only when a surface resists instrumentation.
- **Inference-first classification.** Don't tag markup with `data-aegis-cmd` unless inference genuinely fails. Markup should describe semantics; the recorder should read them.
- **DOM-find replay before iframe replay.** The Compass forms use iframe + postMessage. Newer inline drawers (Pipeline) use direct DOM manipulation. Form Insert/Select/Submit handlers try iframe first, fall through to DOM-find by label. Both must coexist.
- **`page_ready` over heartbeat.** Don't regress nav settle to a fixed delay or heartbeat watch.
- **Schema inventory is authoritative.** Ad-hoc table probing is not acceptable. When in doubt, ask Vaughn for SQL.
- **Orphan cleanup requires explicit numbered options.** When authorization is needed for destructive actions, present numbered choices (1/2/3) with concrete consequences.
